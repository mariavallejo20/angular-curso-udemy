# Project Context

## Purpose
Este proyecto es un entorno de aprendizaje para Angular. El objetivo principal es practicar y asimilar los conceptos del framework Angular siguiendo las secciones de un curso. Cada sección del curso se desarrolla en una rama de git independiente, permitiendo experimentar y versionar el avance de forma ordenada.

## Tech Stack
- Angular
- TypeScript
- Herramientas y dependencias estándar de Angular (RxJS, Jasmine, Karma, etc.)

## Project Conventions

### Code Style
- Seguir las buenas prácticas y convenciones recomendadas por Angular.
- Uso de Prettier y/o ESLint si el curso lo sugiere.
- Nombres de archivos y carpetas en kebab-case.
- Componentes, servicios y módulos nombrados según la guía oficial de Angular.

### Architecture Patterns
- Estructura modular basada en el CLI de Angular.
- Uso de servicios para lógica de negocio y consumo de APIs.
- Lazy loading y separación de módulos si el curso lo cubre.

### Testing Strategy
- Pruebas unitarias con Jasmine y Karma (por defecto en Angular CLI).
- Se agregarán pruebas según lo indique el curso.

### Git Workflow
- Cada sección del curso se desarrolla en una rama independiente (ej: `seccion-01-intro`, `seccion-02-componentes`).
- La rama `main` puede usarse para integrar el avance final o versiones completas.
- Mensajes de commit descriptivos, preferiblemente en español, explicando el propósito del cambio.

## Domain Context
Proyecto educativo orientado al aprendizaje de Angular. No existen restricciones de dominio ni requisitos de negocio.

## Important Constraints
No hay restricciones técnicas, regulatorias ni de negocio. El único objetivo es el aprendizaje.

## External Dependencies
Solo las dependencias estándar de Angular y las que el curso indique.
